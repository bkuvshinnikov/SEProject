# Project Management Approach

## Overview
This project follows a hybrid Scrum/Kanban approach to balance structured planning with flexibility. Scrum principles are used to organize the work into short, iterative cycles (weekly sprints), each with clearly defined goals and deliverables. At the same time, a Kanban board is used to visualize task progress, manage workflow, and adapt to changing priorities throughout the development process.

## Workflow
- Work is planned in weekly sprints
- Tasks are tracked using a Kanban board (To Do → In Progress → Done)
- Priorities may be adjusted dynamically based on progress and findings
- Continuous integration of completed features is preferred over large releases

## Responsibilities

### Developer (Human)
- System architecture and design decisions  
- Core feature implementation  
- Code review and quality assurance  
- Ensuring maintainability and consistency of the codebase  

### AI Assistance
- Generating boilerplate code  
- Writing and suggesting unit tests  
- Assisting with implementation ideas  
- Speeding up routine and repetitive development tasks  

## Goal
This approach aims to maximize development efficiency while maintaining full control over the system’s architecture, code quality, and long-term maintainability.
